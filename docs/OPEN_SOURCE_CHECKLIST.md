# 开源前检查清单

准备把 `qlit-flet/` 单独推到 GitHub 前，建议按这个顺序过一遍。

## 当前公开范围

这次只公开“QLIT养成教育助手”本身：

- GUI
- session 抓取
- session 检测
- 养成教育自动填写

课堂回放相关脚本当前不在这个仓库的公开范围内。

## 必做

1. 补一个正式的 `LICENSE` 文件。  
   没有许可证时，代码默认不是别人可以自由复用的“开源项目”。

2. 检查是否混入本地敏感信息。  
   重点看：
   - `session.txt`
   - AI API Key
   - 导出的设置文件
   - 任何截图、日志、HAR

3. 确认没有把运行产物放进版本库。  
   重点看：
   - `.venv/`
   - `build/`
   - `dist/`
   - `mitmproxy.app/`
   - `__pycache__/`

4. 再看一遍 README。  
   确认别人第一次进入仓库时能看懂：
   - 这是做什么的
   - 怎么运行
   - 有什么边界和风险

## 很建议做

1. 在一台相对干净的机器或新虚拟环境里跑一次启动流程。  
   这样更容易发现“本机上能跑，别人拉下来不行”的隐性依赖。

2. 如果要提供可执行包，用 GitHub Releases，而不是把 `.app` / `.exe` / `.zip` 直接放在仓库里。

3. 如果后面准备接受 PR，最好再补：
   - `CONTRIBUTING.md`
   - issue / PR 模板
   - 一个基础的 smoke test 或检查脚本
